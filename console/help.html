Phineas Help

Introduction

Phineas is a CDC PHINMS ebXML compatible transport system with hooks for syncronous message processing. If that seems like an archane bunch of gibberish... well it probably is. So think about it this way. You have data files. You want to automate file delivery from one place to another. You want to simply put a file into a folder and have it magically appear in a specific folder somewhere else in the big bad internet. That's what Phineas does, but with guaruteed delivery, auditing, logging, security, and full HIPAA compliance.

Navigation

If you made it this far, you must be running the Phineas console. You'll notice a bunch of icons at the top. Click on these to get various functions of the console. The following lists possible activities.

Show the Queues. Phineas tracks all transport activity in what are called queues. Basically these are tables with information about each file sent or received. The entries are color coded so that you can easily see issues. By selecting a record you get a detailed report below the queue list.
Send an ebXML "Ping" to selected route. Use this to test route connectivity and connectivity.
Show the log. The amount and type of information found in the log will vary depending on the Log Level. Beware that if Phineas has been running a long time, this log could be quite large!
Show the configuration. Phineas is configured using a single XML configuration file. The current running configuration is shown. This may be different than the configuration being edited.
Edit Configuration. This is a set of forms that you can use to modify a Phineas configuration, or export a CPA. Configuration parameter are documented below.
Print the console contents (not yet implemented).
Refresh the console and return it to the default function.

Configuration

While Phineas doesn't (yet) have as many configuration parameters as PHINMS, there is still plenty of stuff to tweak. The good news is that generally speaking you won't have to touch much (or any) of it. In fact it's very possible to generate a Phineas distribution that doesn't allow (easy) modification of the configuration.
